# 카카오 알림톡 템플릿 정의서

> COURTSIDE — 테니스 코치 매칭 O2O 서비스
> 카카오 비즈메시지 API 기반 알림톡 템플릿 관리 문서

---

## 1. 발송 정책

```
채널: 카카오 알림톡 (비즈메시지 API)
발신 프로필: COURTSIDE (카카오톡 채널)
발송 조건: NotificationSetting 해당 항목 = true일 때만 발송
대체 발송: 알림톡 실패 시 앱 내 알림(Notification 테이블)으로 대체 저장
변수 표기: #{변수명} — 서버에서 동적 치환
버튼 타입: 웹링크 (앱 딥링크 URL scheme)
```

| 변수 | 출처 | 예시 |
|------|------|------|

**변수**:
| 변수 | 출처 | 예시 |
|------|------|------|

**API 호출 (발송 트리거)**:
```
PUT /api/bookings/:id/hold { reason }
→ Booking.status = ON_HOLD
→ Booking.holdReason = reason (필수)
→ Booking.holdAt = NOW()
→ Notification 레코드 생성 (type: BOOKING_ON_HOLD)
→ 카카오 알림톡 발송
```

## 4. 카카오 알림톡 연동 스펙

### 4.1 API 정보

```
엔드포인트: POST https://api-alimtalk.kakao.com/v2/sender/send
인증: 카카오 비즈메시지 API Key (서버 환경변수)
발신 프로필: 카카오톡 채널 연동 필수
```

### 4.2 템플릿 등록 프로세스

```
1. 카카오 비즈니스 채널 개설
2. 알림톡 발신 프로필 등록
3. 템플릿 등록 (카카오 검수 필요 — 1~3 영업일)
4. 검수 승인 후 templateCode로 발송
```

### 4.3 버튼 타입

| 타입 | 코드 | 용도 |
|------|------|------|
| 웹링크 | WL | 앱/웹 URL 이동 (urlMobile, urlPc) |
| 앱링크 | AL | 앱 딥링크 (iOS scheme, Android scheme) |
| 봇키워드 | BK | 챗봇 키워드 응답 (미사용) |

### 4.4 발송 실패 처리

```
알림톡 발송 실패 시 (번호 오류, 카카오 친구 아님 등):
  → 앱 내 Notification 테이블에 저장 (fallback)
  → 실패 로그 기록 (모니터링용)
  → 재시도 없음 (앱 내 알림으로 대체)
```
